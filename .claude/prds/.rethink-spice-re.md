# Product Brief: spice-re

## Vấn đề (Đã Reframe)

Nhà đầu tư BĐS dòng tiền tại Việt Nam thiếu công cụ giúp tư duy sáng suốt khi ra quyết định mua — không phải thiếu data, mà thiếu cấu trúc tư duy. Bias tâm lý (anchoring vào giá rao, overestimate gross yield, bỏ qua OPEX ẩn) dẫn đến quyết định dựa trên cảm xúc thay vì phân tích có hệ thống. Hiện không có AI agent nào kết hợp cognitive bias detection với cash flow analysis thực tế chạy trên Claude Code.

## Yêu cầu Gốc vs Hướng đi Mới

- **Gốc:** 6 Agent Skills orchestrated (deciding, analyzing-biases, simulating, debating, researching, frameworks) + 20+ Python scripts + SQLite + 8-tool location intelligence — kiến trúc phức tạp, 15 tuần, 4 phases.
- **Reframe:** ONE skill `deciding-real-estate` chứa toàn bộ cognitive loop, gọi 4 core scripts và reference MDs khi cần. Kiến trúc đơn giản, phù hợp Agent Skills standard (mỗi skill triggered độc lập), build nhanh hơn 3x.
- **Insight chính:** Quyết định BĐS thất bại chủ yếu vì tâm lý, không phải thiếu data. "Thinking Coach" (bias detection + structured reasoning + cash flow reality check) mang lại giá trị lớn hơn "Analysis Platform" (thêm tools, thêm data, thêm scripts). Consistency từ cognitive loop là moat — không phải feature count.

## User Mục tiêu

**Chính:** Nhà đầu tư BĐS dòng tiền — ưu tiên thu nhập thụ động ổn định từ cho thuê, đang cần quyết định mua BĐS cụ thể. Có kinh nghiệm nhưng vẫn dễ bị bias khi đối mặt deal hấp dẫn. Dùng Claude Code CLI.

## Kết quả Mong muốn

Sau khi ship, user sẽ NGỪNG phải tra cứu thông tin thủ công và NGỪNG đưa ra quyết định dựa trên cảm xúc. Thay vào đó, mỗi quyết định BĐS sẽ đi qua một structured loop: biases được detect, cash flow được tính thực tế (net yield, không phải gross), assumptions được stress test bởi 5 góc nhìn khác nhau, và output là Decision Card với confidence level rõ ràng. User sẽ CẢM THẤY tự tin hơn vì biết mình đã nghĩ kỹ — không phải vì tool nói "mua" hay "không mua."

## Đặt cược & Rủi ro Chính

**Đặt cược:** Structured cognitive loop + bias detection tạo giá trị vượt trội so với Claude prompting ad-hoc. Consistency là moat.
**Nếu sai:** Tool không mang lại insight mới so với hỏi Claude trực tiếp → giảm complexity, giữ scripts + formulas làm reference library.
**Giảm thiểu:** Test với quyết định BĐS thực của Chef. Nếu Decision Card không thay đổi kết luận so với hỏi Claude không có skill → re-evaluate.

**Rủi ro phụ:**

| Rủi ro | Giảm thiểu |
|--------|-----------|
| SKILL.md quá bloated cho ~5k token limit | Iterate instructions qua 10-15 test cases, split nếu cần |
| Bias detection false positives → mất tin tưởng | Question-based ("Bạn có đang...?") thay vì assert |
| Cash flow formulas sai → mất tiền | Test scripts với known scenarios, cross-check Excel |
| OPEX assumptions sai cho thị trường VN | CASHFLOW-VERIFICATION checklist bắt buộc, conservative defaults |

## Quy mô PRD Đề xuất

**Medium** — Một skill nhưng có depth: cognitive loop 6 bước, 4 scripts cần spec interface rõ ràng, 12 biases cần catalog, 5 personas cần define, formulas cần document. Không phải small (quá nhiều moving parts) nhưng không phải large (chỉ 1 skill, scope bounded).

## Câu hỏi Mở cho PRD Discovery

1. **Token budget allocation:** SKILL.md ~4k tokens — phân bổ thế nào giữa 6 bước cognitive loop? Bước nào cần chi tiết nhất trong instructions vs delegate cho reference MDs?
2. **Script interface contracts:** Exact CLI args, output JSON schema, error handling — cần spec cụ thể trong PRD để implementation không drift.
3. **Debate protocol depth:** 5 personas debate trong 1 turn hay multi-turn? Single turn gọn hơn nhưng multi-turn sâu hơn — tradeoff context window.
4. **Decision archive format:** Markdown file naming convention, metadata fields, how reflection step reads past decisions — cần define để archive có giá trị dài hạn.
5. **Test cases:** Cần 5-7 test scenarios thực tế (mua căn hộ cho thuê, mua nhà trọ, so sánh 2 deals, bán vs giữ, negative cash flow detection) — define trong PRD để QA rõ ràng.
