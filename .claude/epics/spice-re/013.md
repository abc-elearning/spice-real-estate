# bayesian_updater.py — Probability Updating

## Context

When new market evidence emerges (infrastructure announcement, policy change), investors need to systematically update their probability estimates instead of over/underreacting.

## Description

Create `scripts/bayesian_updater.py` implementing Bayesian probability updating with evidence strength mapping, overconfidence detection, and interpretation text.

## Acceptance Criteria

- **FR-6 / Happy path:** `--prior 0.4 --evidence_strength strong --evidence_direction positive` → posterior > 0.4, with interpretation
- **FR-6 / Overconfidence:** If user prior diverges >20% from a base rate → flag in output
- **NFR-2 / Interface:** CLI args → JSON stdout
- **FR-6 / Edge case:** `--prior 0` → error JSON (Bayes undefined at 0)
- **FR-6 / Edge case:** `--prior 1` → error JSON (Bayes undefined at 1)

## Implementation Steps

### Step 1: Create scripts/bayesian_updater.py

- Import from utils: json_output, json_error, validate_range, validate_choice
- Argparse: --prior (float), --evidence_strength (str: weak|moderate|strong|very_strong), --evidence_direction (str: positive|negative), --base_rate (float, optional for overconfidence check)
- Validation: prior strictly between 0 and 1 exclusive (0 < prior < 1)
- Likelihood ratio mapping:
  - weak: 1.5, moderate: 3.0, strong: 6.0, very_strong: 12.0
- If direction=negative: use reciprocal (1/LR)
- Bayes' theorem: posterior = (prior * LR) / (prior * LR + (1 - prior))
- Shift = posterior - prior
- Overconfidence check: if base_rate provided and abs(prior - base_rate) > 0.2 → add overconfidence_warning
- Interpretation text (Vietnamese): describe what the update means

## Tests to Write

- prior=0.5, strong, positive → posterior should be 0.857 (6/(6+1))
- prior=0.4, moderate, negative → posterior < 0.4
- prior=0.0 → error JSON
- prior=1.0 → error JSON
- prior=0.8, base_rate=0.5 → overconfidence warning present
