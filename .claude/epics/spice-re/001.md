## Context

SpiceRE needs a single Agent Skill entry point that orchestrates structured real estate buy/sell/hold decisions through a 6-step cognitive loop. This SKILL.md is the core — everything else (reference files, scripts) exists to support it. It MUST stay under ~5k tokens (Level 2 body limit).

## Description

Create `SKILL.md` with Anthropic Agent Skills frontmatter and a Level 2 body implementing a 6-step cognitive loop: perception, state modeling, analysis, simulation, decision, reflection. Uses directive-style instructions telling Claude when to load reference files (one per step max) and when to run Python scripts via `.venv/bin/python3`. Also set up Python venv with numpy.

## Acceptance Criteria

- [ ] **FR-1 / Happy path:** User asks "Should I buy this apartment for 3.2B VND, rental 13M/month?" → skill triggers, runs all 6 steps, produces Decision Card
- [ ] **FR-1 / Edge case:** User provides incomplete info (missing price) → Step 1 Perception asks max 3 clarifying questions
- [ ] **FR-2 / Partial:** Step 3 Analysis includes instruction to `cat BIASES.md` and scan user input for bias markers
- [ ] **FR-7 / Partial:** Step 5 Decision includes instruction to `cat PERSONAS.md` and run 5-persona debate
- [ ] **FR-8 / Partial:** Step 5 Decision includes instruction to `cat DECISION-CARD.md` and generate Vietnamese Decision Card
- [ ] **NFR-1 / Token budget:** SKILL.md body (after frontmatter) is under 5,000 tokens (`wc -w` × 1.3 < 5000)

## Implementation Steps

### Step 1: Set up Python environment
- Create directory: `mkdir -p .claude/skills/spice-re/deciding-real-estate/scripts`
- Create venv: `python3 -m venv .claude/skills/spice-re/deciding-real-estate/.venv`
- Install numpy: `.venv/bin/pip install numpy`
- Add `.venv/` to `.gitignore` if not already there

### Step 2: Create SKILL.md frontmatter
- Create `.claude/skills/spice-re/deciding-real-estate/SKILL.md`
- Frontmatter (Level 1):
  ```yaml
  ---
  name: deciding-real-estate
  description: >
    Orchestrates structured real estate buy/sell/hold decisions using a 6-step
    cognitive loop (perception, state modeling, analysis, simulation, decision,
    reflection). Triggers when user asks about buying, selling, holding, or
    evaluating real estate properties or investment decisions. Uses bias detection,
    cash flow analysis, Monte Carlo simulation, Bayesian updating, and multi-persona
    debate to produce a structured Decision Card.
  ---
  ```

### Step 3: Write Level 2 body — 6-step cognitive loop
- Each step uses 3-5 directive lines maximum
- **HARD RULE in instructions:** "Load at most ONE reference file per step"
- Step 1 Perception (~400 tokens): Parse intent (buy/sell/hold/compare/research), extract property attributes (location, price, area, type), extract user constraints (budget, timeline, purpose), ask max 3 clarifying questions if critical info missing
- Step 2 State Modeling (~500 tokens): Build structured markdown models — User Profile (finances, risk tolerance, horizon), Property Model (attributes, comparable context), Market Context (cycle phase, macro), Constraint Map (hard vs soft)
- Step 3 Analysis (~800 tokens): `cat BIASES.md` → scan user input against detection markers, report biases as questions. Framework selection logic: cash flow intent → DCF+DSCR path; capital gain → EV+Monte Carlo; sell/hold → Prospect Theory+Regret Minimization; comparison → Decision Matrix. Optionally `cat SOURCES-VN.md` → web search for macro data
- Step 4 Simulation (~600 tokens): Run scripts via `.venv/bin/python3 scripts/{name}.py --args`. Cash flow path: run `cashflow_proforma.py` + `dcf_calculator.py`. Capital gain: run `monte_carlo.py`. New evidence: run `bayesian_updater.py`. Parse JSON output, present key metrics
- Step 5 Decision (~800 tokens): `cat PERSONAS.md` → run 5-persona debate protocol (each persona 3-5 sentences, identify tensions, synthesize). `cat DECISION-CARD.md` → generate Vietnamese Decision Card with all sections filled. Confidence scoring: Low <40%, Medium 40-70%, High >70%
- Step 6 Reflection (~400 tokens): Save Decision Card to `.spice_re/decisions/YYYY-MM-DD-{slug}.md` (`mkdir -p` first). Compare with past decisions if archive exists. Suggest review timeline. Always end with disclaimer.
- Include framework selection logic block
- Include disclaimer text

- If over budget: compress instructions, remove redundancy, use shorter directive phrases
